Laravel 8 !

Le 01/10/2021
Larry Dimmick
Laravel 8 est maintenant publié et comprend de nombreuses nouvelles fonctionnalités, notamment Laravel Jetstream, un répertoire de modèles, des classes d'usine de modèles, un écrasement de la migration, des améliorations de limitation du débit, des assistants de test temporel, des composants de lame dynamiques et bien d'autres fonctionnalités. Avant de nous lancer dans les nouvelles fonctionnalités, nous aimerions souligner qu'à partir de la version 6, Laravel suit désormais semver et publiera une nouvelle version majeure tous les six mois. Vous pouvez voir comment fonctionne le processus de publication ici.
Laravel Jetstream améliore l'échafaudage d'interface utilisateur Laravel existant dans les versions précédentes. Il fournit un point de départ pour de nouveaux projets, y compris la connexion, l'enregistrement, la vérification des e-mails, l'authentification à deux facteurs, la gestion de session, la prise en charge des API via Laravel et la gestion d'équipe.
Le squelette d'application de Laravel 8 comprend un répertoire app/Models. Toutes les commandes de générateur supposent que des modèles existent dans app/Models ; Cependant, si ce répertoire n'existe pas, le framework supposera que l'application conserve les modèles dans le dossier app/.
Si votre application contient de nombreux fichiers de migration, vous pouvez désormais les écraser dans un seul fichier SQL. Ce fichier sera exécuté en premier lors de l'exécution des migrations, suivi de tous les fichiers de migration restants qui ne font pas partie du fichier de schéma écrasé. L'écrasement des migrations existantes peut réduire le gonflement des fichiers de migration et éventuellement améliorer les performances lors de l'exécution des tests.
